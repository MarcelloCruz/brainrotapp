1. **Read the Brain First:** Before answering any prompt or writing code, you MUST read `brain.md` to understand the current state of the project.
2. **Update the Brain Last:** After completing a feature or fixing a bug, you MUST update `brain.md` to reflect the new state.
3. **Architecture & State Management:** * Use [Insert your preferred state management here, e.g., Riverpod or Provider].
    * Keep UI completely separated from business logic.
4. **Native Bridging:** We are relying heavily on Android and iOS native APIs. Always create clean `MethodChannel` interfaces in Dart. When modifying native code (Kotlin/Swift), explain the changes clearly before writing them.
5. **No Destructive Overwrites:** Never delete working native code to try a "new approach" without explicit permission.
